CampCart Developer Hub

Welcome to the official developer guide for the CampCart backend integration. Build, manage, and scale the ultimate e-commerce platform across users, vendors, and delivery agents.

API Server Base URL
https://campcart-backend.onrender.com

💡 Frontend Integration Requirement: Make sure your requests include the Content-Type: application/json header. Enable credentials flag (e.g. withCredentials: true in Axios) to allow the backend to send and read cookies if implemented.

Authentication Routes
POST /api/v1/signup Create user ▼

Registers a new user account in the CampCart database. The role dictates access levels across the platform (user, vendor, admin, deliveryAgent).

Request Body Fields
Field Type Requirement Description
email string Required A valid email address for account credentials.
password string Required A secure password (hashed via bcrypt).
fullName string Required The full name of the user.
role string Required Must be user, vendor, admin, or deliveryAgent.
Request Payload Example
JSON 
"email": "john@example.com",
"password": "securePass123!",
"fullName": "John Doe",
"role": "user"
Success Response Example 201 Created
"message": "Account created successfully",
"role": "user"
Error Responses
400 Bad Request Missing email, fullName, password or role.
{"message": "Email, full name,password or role not provided"}
403 Forbidden User with this email already exists.
{"message": "User with this info already exists"}
500 Internal Server Error An unexpected server error occurred during creation.
{"message": "An error occurred", "error": "..."}
POST /api/v1/login Retrieve access token ▼

Authenticates credentials and establishes a session for the user.

Request Body Fields
Field Type Requirement Description
email string Required The registered account email.
password string Required The user password.
Request Payload Example
JSON 
"email": "john@example.com",
"password": "securePass123!"
Success Response Example 200 OK
"message": "Login successful",
"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
Error Responses
404 Not Found No registered user matches the provided email.
{"message": "User not found"}
401 Unauthorized Password validation comparison failed.
{"message": "Incorrect password"}
User Profile Services
GET SECURE /api/v1/auth/me Get active user ▼

Retrieves the verified user data for the active session.

Required Request Configuration
// Client must send cookies
"credentials": "include"
Success Response Example 200 OK
"message": "Information pulled",
"info": {
  "_id": "64f8a2c1b9...",
  "fullName": "John Doe",
  "email": "john@example.com",
  "role": "user",
  "status": "active"
}
Error Responses
401 Unauthorized Authorization cookie is invalid or missing.
{"message": "No access token"}
404 Not Found The user matching the token was not found.
{"message": "User with this info does not exist"}
POST SECURE /api/v1/users/profile Update user details ▼

Updates the phone number, address, and business name for the logged-in user or vendor.

Required Request Configuration
// Client must send cookies
"credentials": "include"
Request Body Fields
Field Type Requirement Description
phoneNumber string Optional User's contact number.
address string Optional User's delivery or store address.
businessName string Optional Store name (only applies to vendors).
Request Payload Example
JSON 
"phoneNumber": "+1234567890",
"address": "123 Vendor St, Market City",
"businessName": "Jane's Goods"
Success Response Example 200 OK
"message": "Info added successfully",
"info": {
  "_id": "64f8a2c1b9...",
  "phoneNumber": "+1234567890",
  "address": "123 Vendor St, Market City",
  "businessName": "Jane's Goods"
}
Vendor Services
POST VENDOR /api/v1/vendor/products Create a new product ▼

Allows an authenticated vendor to create a new product and add it to the platform catalog.

Required Request Configuration
// Client must send cookies
"credentials": "include"
Request Body Fields
Field Type Requirement Description
name string Required The name of the product.
description string Required Detailed product description.
price number Required Product price in the platform's currency.
category string (ObjectId) Required The ID of the category this product belongs to.
images array of strings Optional List of URLs to the product images.
stock number Optional Initial stock level (defaults to 0).
Request Payload Example
JSON 
"name": "Camping Tent",
"description": "A durable 4-person camping tent.",
"price": 129.99,
"category": "64f8b2d1c...",
"images": ["https://example.com/tent1.jpg"],
"stock": 50
Success Response Example 201 Created
"message": "Product created successfully",
"product": {
  "_id": "64f9c3d2e...",
  "name": "Camping Tent",
  "price": 129.99,
  "vendor": "64f8a2c1b..."
}
GET VENDOR /api/v1/vendor/products Get vendor products (Paginated) ▼

Retrieves a paginated list of products created by the authenticated vendor.

Required Request Configuration
// Client must send cookies
"credentials": "include"
Query Parameters
Parameter Type Requirement Description
page number Optional Page number to retrieve (default: 1).
limit number Optional Number of items per page (default: 10).
Request Example
GET /api/v1/vendor/products?page=1&limit=10
Success Response Example 200 OK
"message": "product pulled",
"products": [
  {
    "_id": "64f9c3d2e...",
    "name": "Camping Tent",
    "price": 129.99,
    "vendor": "64f8a2c1b..."
  }
],
"currentPage": 1,
"totalPages": 5,
"totalProducts": 48
